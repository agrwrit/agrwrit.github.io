Error AADSTS50076 in Cayosoft Administrator Web Portal

Environment

  • The authentication method you set up in the Web Portal is Automatic SSO for Azure Active Directory / Microsoft 365 accounts.

Symptom

When a user attempts to log in, Cayosoft Administrator prompts the following error to them:

AADSTS50076: Due to a configuration change made by your administrator, or because you moved to a new location, 
you must use multi-factor authentication to access

Cause

The user was marked as risky by Entra ID preventing their logon.

NOTE: To diagnose the cause, navigate to ID Protection > Risky users in your Entra admin center. Review the list of risky users and confirm whether the affected user is on the list.

Resolution

Dismiss the user risk or confirm the user is safe:

  1. In your Entra admin center, navigate to ID Protection > Risky users.

  2. Highlight the affected user and click Dismiss user(s) risk or Confirm user(s) safe. Confirm the selection.

  3. Try logging in the Web Portal to verify the user is not considered risky.
